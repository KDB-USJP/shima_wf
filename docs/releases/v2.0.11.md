# Shima v2.0.11: Precision Sliced Ecosystem & Sync Overhaul 🏝️🛡️

This release introduces the **Shima Sliced** ecosystem for VRAM-efficient high-resolution processing and significant stability improvements to node synchronization and dependency scanning.

## 🚀 Key Improvements

### 1. Shima Sliced Ecosystem (NEW)
- **Precision Slicing**: Implemented `replicate` padding in the **Sliced Upscaler** to eliminate thin black lines at slice boundaries.
- **Orientation Clarity**: Standardized orientation logic: `Vertical` now creates columns and `Horizontal` creates rows.
- **Modular Hubs**: 
    - **Sliced Image Hub**: Splits upscaled batches for individual node processing.
    - **Slice Combiner Hub**: Re-merges processed segments with feathered alpha-blending.
    - **Sliced Commons**: Central source of truth for all slicing parameters.

### 2. Node Synchronization & Stability
- **Photo Remix Sync**: Renamed `seed` to `s33d` to align with the Shima standard. Now supports full **CommonParams** synchronization with a restore of the Topbar toggle.
- **ControlNet "Skip" Mode**: Added a `skip` fit method to the **ControlNet Agent**, allowing pre-processed control maps to pass through at their original resolution.
- **Deep Dependency Scanning**: The **Dependency Generator** now correctly identifies standard ComfyUI loaders (PNG, JPG, etc.) and features expanded extension support for videos and masks.

### 3. Documentation & UI
- **Full Reference Update**: Updated the Shima Node Reference with detailed documentation for all new Sliced nodes and enhanced "Used Values" transparency across the ecosystem.
