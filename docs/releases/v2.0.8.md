# Shima v2.0.8: Registry Fixes & Node Standardizing 🛠️🏷️

This release focuses on backend stability, standardizing node naming for better workflow compatibility, and hardening the Shima Hub registry.

## 🚀 Highlights

### 1. Node Standardizing
- **ControlNet Agent**: Renamed "Control Agent" to "ControlNet Agent" across all internal definitions and the frontend UI to ensure consistency with industry standards.
- **Title Sync**: Updated existing workflows and panel definitions to reflect the new naming convention.

### 2. Registry & Security Hardening
- **Publisher ID Casing**: Fixed a critical issue where inconsistent casing for Publisher IDs could cause registry lookup failures.
- **Safe Loader Logic**: Enhanced the model loader to handle unexpected `app` object states in older ComfyUI versions.
- **Authenticated Downloads**: Added explicit tracking for `repo_id` and `subfolder` in the background downloader to support more complex HuggingFace repository structures.

### 3. UI/UX Refinement
- **Hub Status Reporting**: Improved the reliability of the "Installed" status check by adding robust path validation for specific model families (Flux, SDXL, SD1.5).
