# Shima v2.0.9: Subgraph Compatibility & Robustness 🕸️🛡️

A major architectural update ensuring Shima nodes function flawlessly within nested graph structures (subgraphs) while hardening the extension against edge-case traversal errors.

## 🚀 Highlights

### 1. Full Subgraph Compatibility
- **Local Graph Context**: Conducted a comprehensive audit and refactoring of all Shima JS files (`shima_styler.js`, `shima_panels.js`, `shima_omnijog.js`, etc.) to replace hardcoded `app.graph` references with local `node.graph` contexts.
- **Nested Search Logic**: Updated all recursive node search functions to correctly traverse subgraphs without leaking into the root graph context inappropriately.

### 2. Performance & Stability Hardening
- **Recursion Depth Limits**: Implemented safety depth limits (max 10) and circular reference checks for all recursive graph traversal logic in `shima_checker.js`, `noodman.js`, and `shima_control_panel.js`. This prevents potential browser freezes in infinitely complex or circular workflows.
- **Checker Timeout**: Added a 5-second connection timeout to all `fetchApi` calls within the Workflow Checker to prevent the "stuck in scanning" bug when external assets or seeds are slow to respond.

### 3. Hub Node URL Refinement
- **Verified Download Links**: Migrated model download URLs to verified HuggingFace and Civitai endpoints for "Absolute Reality", "Juggernaut XL", and "Perfect Hands".
- **Lumina Cleanup**: Removed the experimental Lumina model and LoRA from the official dependencies list to focus on current stable standards.
