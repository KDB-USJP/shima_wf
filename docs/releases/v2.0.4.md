# Release Notes - Shima v2.0.4

This hotfix release addresses critical usability feedback regarding workflow persistence and adds more flexibility to the control system.

## Key Improvements

### 1. Persistent Workflow Connectivity 🛡️🔗
We've fixed an issue where manual overrides to **Use Everywhere (UE)** connection properties were being reset on every reload.
- **Persistence Guaranteed**: If you manually enable/disable UE for a specific input, your setting will now persist across saves and reloads.
- **Master Prompt BNDLs**: All BNDL inputs on the Master Prompt now allow UE connections by default for better "Out of the Box" routing.

### 2. Restored "Pin to Control Panel" Menu 🖱️⌨️
The context menu option to pin widgets to the **Shima Control Panel** has been refactored for maximum reliability.
- **Prototype Integration**: Uses a robust injection method that ensures the menu appears on all Shima nodes, even with heavy 3rd party extension interference.
- **Universal Pinning**: Reliable detection of all widgets, including internal Sampler params.

### 3. Interactive Crop: Active Toggle ✂️
Added an optional `active` input to the **Shima Interactive Crop** node.
- **Flexible Bypass**: Accepts Boolean or Integer (0/1) signals to dynamically enable or disable cropping in your UI chain.
- **Graceful Failover**: When inactive, the node returns the full-size image, preventing broken links downstream.
